Statistics Functions

Statistical functions that operate on multiple arrays.

sfMedianArrays

Computes the median of values from multiple AFL arrays, bar by bar.

The arrays are accessed by name using the pattern prefix + number

(e.g. prefix1, prefix2, …).

For bars where all input values are empty, the result is Empty.

For an even number of non-empty values, the average of the two middle

values is returned.

Syntax: sfMedianArrays( prefix, count )

Arg Type Description
prefix string Base name of the AFL arrays (e.g. "MyArray" for arrays MyArray1, MyArray2, …)
count float Number of arrays to read (1‑based). Must be ≥ 1

Returns: array — Median of the input values at each bar.

Example:


// Assume MyArray1, MyArray2, MyArray3 exist

Med = sfMedianArrays( "MyArray", 3 );

sfMedianArray2

Optimized median of exactly 2 arrays. For each bar, returns the average

of the two values, or the single non-empty value if one is empty,

or Empty if both are empty.

Syntax: sfMedianArray2( Array1, Array2 )

Arg Type Description
Array1 array First input array
Array2 array Second input array

Returns: array — Bar-by-bar median of the two arrays.

sfMedianArray4

Optimized median of exactly 4 arrays.

Syntax: sfMedianArray4( Array1, Array2, Array3, Array4 )

Arg Type Description
Array1‑Array4 array Four input arrays

Returns: array — Bar-by-bar median. For an even number of non-empty

values at a bar, the average of the two middle values is returned.

sfMedianArray6

Optimized median of exactly 6 arrays.

Syntax: sfMedianArray6( Array1, Array2, Array3, Array4, Array5, Array6 )

Arg Type Description
Array1‑Array6 array Six input arrays

Returns: array — Bar-by-bar median.

sfPeriodicStat

Computes a sliding-window statistic (mean, cumulative sum, or median)

per period group. Each bar belongs to a group indicated by PeriodNumber

(e.g. day of week 1–7, month 1–12, etc.). A separate sliding window of

the last n values is maintained for each unique period number, and the

chosen statistic is computed over that window.

Syntax: sfPeriodicStat( Values, PeriodNumber, n, type )

Arg Type Description
Values array The input values
PeriodNumber array Period identifier for each bar (e.g. DayOfWeek(), Month())
n float (optional) Sliding window size. Default: 20
type float (optional) Statistic type: 0 = mean, 1 = sum (cumul), 2 = median. Default: 0

Returns: array — The per-bar statistic for the group to which each bar

belongs.

Example:


// Average volume for each weekday over the last 20 weeks

AvgVol = sfPeriodicStat( Volume, DayOfWeek(), 20, 0 );

The first bar of each period group returns the average/sum/median of

all values encountered so far (results are valid from the first occurrence).

sfRegularize

Fills gaps (Empty values) in a time series array to produce a

regularized (gap‑free) series.

Syntax: sfRegularize( A, opt )

Arg Type Description
A array Input time series (may contain Empty gaps)
opt float (optional) Filling method. Default: 0
opt Behavior
0 **Forward‑fill** — copy the last valid value forward
≥ 1 **EMA** — fill with an exponential moving average of the last opt valid values (α = 2/(opt+1))
−1 **Median‑2** — fill with the median (average) of the two most recent valid values
< −1 **Linear regression** — fill by extrapolating a straight line fitted to the last −opt valid values

Returns: array — The regularized series (same length as input).

Notes:

  • All filling methods require at least one prior valid value; bars before

    • the first valid value remain Empty.

  • For `opt ≥ 1`, the minimum effective period is 2.
  • For `opt < −1`, regression requires at least 2 valid points.

    • sfFirstValue

    Returns the first non‑Empty value in the array.

    Syntax: sfFirstValue( A )

    Arg Type Description
    A array Input array

    Returns: float — The first valid value, or Empty if all values are empty.

    sfDiff

    Computes successive differences between array values. The first valid bar

    is set to Empty (no predecessor to diff against). For opt > 1, computes A[n] − A[n − opt].

    Syntax: sfDiff( A, opt )

    Arg Type Description
    A array Input array
    opt float (optional) Lag. Default: 1 (consecutive bars)

    Returns: array — Differences. Empty where either value is missing.

    sfTestAverage

    Sliding‑window one‑sample t‑test for the mean. At each bar i, tests

    the null hypothesis H₀ that the mean of the last n values of X

    equals A[i]. The alternative depends on the tail parameter.

    The test statistic is t = (x̄ − A[i]) / (s / √count) where x̄ is

    the sample mean, s the sample standard deviation, and count the

    number of non‑Empty values in the window. Under H₀, t follows a

    Student t‑distribution with count‑1 degrees of freedom. The

    p‑value is computed via the regularized incomplete beta function

    (Lentz's continued fraction).

    Syntax: sfTestAverage( X, A [, n [, level [, tail ]]] )

    Arg Type Description
    X array Input values
    A array Hypothesized mean (one value per bar)
    n float (optional) Window size. Default: 50
    level float (optional) Significance level α. Default: 0.05
    tail float (optional) Test direction: 0 = two‑sided, 1 = one‑sided in observed direction. Default: 0

    Returns: array — 1.0 if H₀ is rejected at the given level,

    0.0 otherwise, Empty if the window is not yet full or fewer

    than 2 non‑Empty values are available.

    Notes:

  • Two‑sided test (`tail=0`): rejects H₀ when the p‑value is less

    • than level. Equivalent to testing x̄ ≠ A[i].

  • One‑sided test (`tail=1`): tests in the direction of the observed

    • deviation (x̄ > A[i] or x̄ < A[i]). The one‑sided p‑value is

    exactly half the two‑sided p‑value, so the one‑sided test has

    greater power to detect a deviation in that direction.

  • When all values in the window are equal (zero variance), H₀ is

    • rejected if level > 0 and the mean differs from A[i].

    Example (two‑sided):

    
// Test if the 20‑bar rolling mean of Close differs from 50

result = sfTestAverage( Close, 50, 20, 0.05, 0 );

    Example (one‑sided):

    
// Test if the 20‑bar rolling mean of Close is significantly above 50

result = sfTestAverage( Close, 50, 20, 0.05, 1 );

    sfTestThreshold

    Sliding‑window binomial exact test for a proportion. At each bar i,

    tests whether the proportion of values in X exceeding threshold

    over the last n bars is at least p0. The alternative depends on

    the tail parameter.

    The test uses the binomial distribution Binom(m, p0) where m is

    the number of non‑Empty values in the window and k the count

    that exceed threshold. The p‑value is computed via the regularized

    incomplete beta function (Lentz's continued fraction), which gives

    the exact binomial tail probability.

    Syntax: sfTestThreshold( X, threshold [, p0 [, n [, level [, tail ]]]] )

    Arg Type Description
    X array Input values
    threshold float Cutoff value that defines "success" (X > threshold)
    p0 float (optional) Hypothesized proportion under H₀. Default: 0.5
    n float (optional) Window size. Default: 50
    level float (optional) Significance level α. Default: 0.05
    tail float (optional) Test direction: 0 = lower‑tailed (proportion < p0), 1 = upper‑tailed (proportion > p0). Default: 1

    Returns: array — 1.0 if H₀ is rejected at the given level,

    0.0 otherwise, Empty if the window is not yet full or fewer

    than 1 non‑Empty value is available.

    Notes:

  • Upper‑tailed test (`tail=1`): rejects H₀ when the proportion

    • of successes significantly exceeds p0.

  • Lower‑tailed test (`tail=0`): rejects H₀ when the proportion

    • of successes is significantly below p0.

  • The test is exact (not a normal approximation), so it works

    • reliably even for small windows and extreme proportions.

  • Use `threshold=0.5` with binary signals (0/1 values) to test

    • if the win rate exceeds p0.

    Example (upper‑tailed):

    
// Test if the 20‑bar win rate exceeds 60%

result = sfTestThreshold( BuySignal, 0.5, 0.6, 20, 0.05, 1 );

    Example (lower‑tailed):

    
// Test if the 20‑bar win rate is below 40%

result = sfTestThreshold( BuySignal, 0.5, 0.4, 20, 0.05, 0 );

    sfHitProbability

    Sliding‑window probability that prices reach level A (upper target)

    before level B (lower target), based on a Brownian motion with drift

    estimated from recent log‑returns.

    Two methods are available:

  • **method=0** (default): Analytical Brownian motion formula using

    • drift μ and volatility σ estimated from the last n log‑returns

    of price P. The probability is:

    `math

    P(hit A before B) = (1 − exp(−θ·(log(P)−log(B)))) / (1 − exp(−θ·(log(A)−log(B))))

    `

    where θ = 2μ/σ². When the drift is near zero, this reduces to the

    position‑only formula. When variance is negligible, the result is

    driven entirely by the sign of the drift.

  • **method=1**: Position‑only formula that ignores drift:

    • `math

    P = (log(P) − log(B)) / (log(A) − log(B))

    `

    Syntax: sfHitProbability( P, A, B [, n [, method ]] )

    Arg Type Description
    P array Price series
    A array Upper target level (take‑profit)
    B array Lower target level (stop‑loss)
    n float (optional) Window size for drift/vol estimation. Default: 50
    method float (optional) 0 = analytical with drift, 1 = position‑only. Default: 0

    Returns: array — probability in [0, 1], Empty if the window is

    not yet full or if A[i] ≤ B[i].

    Notes:

  • The formula assumes log‑prices follow a Brownian motion with

    • constant drift and volatility over the estimation window.

  • `method=0` requires at least 3 bars of data (n ≥ 3) to estimate

    • both drift and variance.

  • `method=1` works with any window size n ≥ 1 and does not require

    • return history.

  • When `P[i] ≥ A[i]` the probability is `1.0` (already at or above

    • the upper target). When P[i] ≤ B[i] it is 0.0.

  • When `A[i] ≤ B[i]` the barriers are invalid and the result is

    • Empty.

    Example (analytical):

    
// Probability that Close reaches 110 before 90 (20‑bar window)

result = sfHitProbability( Close, 110, 90, 20, 0 );

    Example (position‑only):

    
// Simple position‑based probability

result = sfHitProbability( Close, 110, 90, 20, 1 );

    sfHitLevel

    Probability that prices reach or exceed a target level within a given horizon,

    using Brownian motion with drift estimated from a sliding window of log-returns.

    Syntax: sfHitLevel( Price, level [, est_n [, horizon ]] )

    Arg Type Description
    Price array Price series
    level float Target level
    est_n float (optional) Estimation window for drift/vol (default 50)
    horizon float (optional) Forward horizon in bars (default = est_n)

    Returns: array — Probability for each bar (0–1), EMPTY_VAL until est_n bars of history are available.

    Note: The formula uses the reflection principle for Brownian motion with drift:

    P(hit) = Φ((μ−σ²/2)·T − d)/(σ√T) + exp(2·(μ−σ²/2)·d/σ²)·Φ((−(μ−σ²/2)·T − d)/(σ√T))

    where d = log(level/Price), μ and σ are estimated from the trailing window, and T = horizon. If level ≤ Price, probability is 1 immediately.

    Example:

    
// Probability Close reaches 120 within 20 bars (est_n=50, horizon=20)

prob = sfHitLevel( Close, 120, 50, 20 );

    sfHitMovingTarget

    Probability that prices reach or exceed a moving target level within a given

    horizon, using Brownian motion with drift estimated from a sliding window

    of log-returns. The target is an array, allowing bar-by-bar varying levels.

    Syntax: sfHitMovingTarget( Price, Target [, est_n [, horizon ]] )

    Arg Type Description
    Price array Price series
    Target array Target level (may vary per bar)
    est_n float (optional) Estimation window for drift/vol (default 50)
    horizon float (optional) Forward horizon in bars (default = est_n)

    Returns: array — Probability for each bar (0–1), EMPTY_VAL until est_n bars of history are available, or when Target is empty.

    Example:

    
// Probability Close reaches 10-point trailing target within 20 bars

target = Close + 10;

prob = sfHitMovingTarget( Close, target, 50, 20 );
