Store Functions

Key-value store — a map of named AmiVar values (float, string, or null),

identified by a unique numeric ID. Created via sfStoreCreate, accessed

by key, and destroyed via sfStoreDestroy.

All functions return Null on invalid input (bad handle).

sfStoreCreate

Creates a new empty store.

Syntax: sfStoreCreate( size )

Arg Type Description
size float Ignored (reserved for future use)

Returns: float — The store ID.

sfStoreDestroy

Destroys a store and frees all associated memory.

Syntax: sfStoreDestroy( id )

Arg Type Description
id float Store ID to destroy

Returns: Nothing

sfStoreSize

Returns the number of key-value pairs in the store.

Syntax: sfStoreSize( id )

Arg Type Description
id float Store ID

Returns: float — The number of entries.

sfStoreGet

Retrieves the value at a given key.

Syntax: sfStoreGet( key, id )

Arg Type Description
key string The key to look up
id float Store ID

Returns: AmiVar — The value (float, string, or null if not found).

sfStoreSetFloat

Sets a float value at the given key and returns the old value.

If the key does not exist, it is created and Null is returned.

Syntax: sfStoreSetFloat( key, value, id )

Arg Type Description
key string The key to set
value float The new float value
id float Store ID

Returns: AmiVar — The previous value, or null if the key was created.

sfStoreSetString

Sets a string value at the given key and returns the old value.

If the key does not exist, it is created and Null is returned.

Syntax: sfStoreSetString( key, value, id )

Arg Type Description
key string The key to set
value string The new string value
id float Store ID

Returns: AmiVar — The previous value, or null if the key was created.

sfStoreClear

Removes all key-value pairs from the store.

Syntax: sfStoreClear( id )

Arg Type Description
id float Store ID

Returns: Nothing

sfStoreToString

Serializes the store to a JSON-like string.

Syntax: sfStoreToString( id )

Arg Type Description
id float Store ID

Returns: string — e.g. {"age":30,"name":"hello"}.

sfStoreFromString

Parses a JSON-like string into the store. If id is -1, a new store is

created. If id is a valid store ID, the existing store is cleared and

reloaded.

Syntax: sfStoreFromString( str, id )

Arg Type Description
str string The JSON-like string to parse
id float Store ID (or -1 to create new)

Returns: float — The store ID, or null if parsing fails.

Store Example


// Create a store

s = sfStoreCreate(0);



// Load data from string

sfStoreFromString("{\"name\":\"AAPL\",\"price\":150.25,\"active\":null}", s);



// Read values

name = sfStoreGet("name", s);    // "AAPL"

price = sfStoreGet("price", s);  // 150.25

active = sfStoreGet("active", s); // null (IsNull(active) == 1)



// SetFloat creates the key if it doesn't exist

old = sfStoreSetFloat("volume", 1000000, s); // Null (key was created)

old = sfStoreSetFloat("price", 151.00, s);   // 150.25 (old value)



// SetString creates keys and returns old value

old = sfStoreSetString("sector", "Technology", s); // Null (key was created)

old = sfStoreSetFloat("price", 152.00, s);        // 151.00 (old value)



// Clear all

sfStoreClear(s);



// Destroy

sfStoreDestroy(s);
