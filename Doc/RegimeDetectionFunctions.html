Regime Detection Functions

sfMarketRegime

Detects the market regime at each bar by combining direction, volatility, and momentum/consistency signals into a single integer.

The function modifies three output arrays in place with the individual metrics, and returns the combined regime as a new array.

Syntax: sfMarketRegime( Price, outDir, outVol, outMom, period, mode )

Arg Type Description
Price array Input price array
outDir array *(modified)* Direction score: −1 (bear) to +1 (bull)
outVol array *(modified)* Volatility score: 0 (calm) to 1 (volatile)
outMom array *(modified)* Mode 0: normalized ROC. Mode 1: trending score −1 to +1
period float Sliding window size (minimum 10, default 20)
mode float 0 = ROC-based momentum, 1 = Hurst-based consistency

Returns: array — Regime integer per bar:

Value Regime Conditions
+2 Strong bull Direction ≥ 0.5, Volatility ≥ 0.7, \ Momentum\ ≥ 0.3
+1 Weak bull Direction > 0.3, not strong bear/bull
0 Neutral \ Direction\ ≤ 0.3
−1 Weak bear Direction < −0.3, not strong bear/bull
−2 Strong bear Direction ≤ −0.5, Volatility ≥ 0.7, \ Momentum\ ≥ 0.3

Details:

  • **Direction**: `(Price − EMA) / (rolling std-dev)`, clamped to [−1, +1].
  • **Volatility**: Percentile rank of rolling standard deviation within the window, mapped to [0, 1].
  • **Mode 0 — Momentum**: Rate of change of price over `period` bars, normalized by volatility.
  • **Mode 1 — Consistency**: Hurst exponent (DFA‑1) estimated over a 20‑bar sub‑window. Values > 0.6 indicate trending (+1), < 0.4 indicate mean‑reverting (−1), and in‑between maps linearly to [−1, +1].

    • Example:

    
Dir = 0; Vol = 0; Mom = 0;

Regime = sfMarketRegime( Close, Dir, Vol, Mom, 20, 0 );



// Plot signals

Plot( Regime == 2, "Strong Bull", colorGreen, styleLine );

Plot( Regime == 0, "Neutral",     colorGray,  styleLine );

Plot( Regime == -2,"Strong Bear", colorRed,   styleLine );
