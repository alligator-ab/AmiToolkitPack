sfBSCall

Calculates the price of a European call option using the Black-Scholes model.

Syntax: sfBSCall( S, K, T, r, q, sigma )

Arg Type Description
S float Spot price
K float Strike price
T float Time to maturity (years)
r float Risk-free rate
q float Continuous dividend yield
sigma float Volatility

Returns: float — Call option price. Returns Null if any parameter is non-positive.

Example:


price = sfBSCall( 100, 100, 1, 0.05, 0.02, 0.20 );

sfBAWCall

Calculates the price of an American call option on a spot underlying using the Barone-Adesi & Whaley (1987) quadratic approximation.

Syntax: sfBAWCall( S, K, T, r, q, sigma )

Arg Type Description
S float Spot price
K float Strike price
T float Time to maturity (years)
r float Risk-free rate
q float Continuous dividend yield
sigma float Volatility

Returns: float — American call option price. Returns Null if any required parameter is non-positive.

Notes:

  • When `q ≤ 0` (no dividends), early exercise is never optimal for an American call, so the price equals the European `sfBSCall` price.
  • The BAW method uses a critical asset price `S*` that separates the hold and immediate-exercise regions, found via binary search.

    • Example:

    
// ATM American call, 1 year, 5% rate, 2% dividend, 20% vol

price = sfBAWCall( 100, 100, 1, 0.05, 0.02, 0.20 );

    sfSABRIV

    Computes the SABR (Hagan et al. 2002) closed-form implied volatility for a given set of model parameters, forward, strike, and time to maturity.

    Syntax: sfSABRIV( alpha, beta, rho, nu, F, K, T )

    Arg Type Description
    alpha float SABR volatility parameter (α > 0)
    beta float SABR shape parameter (0 ≤ β ≤ 1)
    rho float SABR correlation parameter (-1 ≤ ρ ≤ 1)
    nu float SABR vol-of-vol parameter (ν ≥ 0)
    F float Forward price
    K float Strike price
    T float Time to maturity (years)

    Returns: float — SABR implied volatility. Returns Null if any required parameter is invalid.

    Notes:

  • Implements the Hagan et al. (2002) asymptotic expansion for the SABR implied volatility.
  • The ATM case (F ≈ K) uses the simpler ATM formula; the general case uses the full expansion with the ζ/x(ζ) transformation.
  • For ζ near zero a Taylor expansion is used to avoid division by zero.
  • Parameter ranges are clamped: β → [0, 1], ρ → [−0.999, 0.999]; α and ν are floored at 1e-14.

    • Example:

    
// Compute SABR IV for an ATM option

iv = sfSABRIV( 0.2, 0.5, 0.0, 0.1, 100, 100, 1 );

    sfSABRCalibrate

    Calibrates the SABR model parameters (α, ρ, ν) to a set of market implied volatilities at different strikes via Nelder-Mead optimisation. The β parameter is fixed by the user.

    Syntax: sfSABRCalibrate( strikes[], ivs[], F, T, beta )

    Arg Type Description
    strikes[] array Array of strike prices (must have at least 3 valid entries)
    ivs[] array Array of market implied volatilities at corresponding strikes
    F float Forward price
    T float Time to maturity (years)
    beta float Fixed SABR shape parameter (0 ≤ β ≤ 1)

    Returns: string — Comma-separated values "α,ρ,ν" on success, or Null if calibration fails (insufficient data, invalid parameters, or no convergence).

    Notes:

  • The arrays `strikes[]` and `ivs[]` are read until the first empty value (EMPTY_VAL = −1e10). Elements where either strike or IV is non-positive are skipped.
  • At least 3 valid (positive, non-empty) data points are required for calibration.
  • The Nelder-Mead simplex search is bounded: α > 0, ρ ∈ [−1, 1], ν > 0.
  • Use `StrExtract()` to parse the returned string.

    • Example:

    
strikes = [ 90, 95, 100, 105, 110 ];

ivs     = [ 0.28, 0.25, 0.22, 0.20, 0.19 ];

result  = sfSABRCalibrate( strikes, ivs, 100, 1, 0.5 );

alpha = StrExtract( result, 0 );

rho   = StrExtract( result, 1 );

nu    = StrExtract( result, 2 );

    sfSABRAmericanCall

    Prices an American call option on a spot underlying by combining the SABR (Hagan et al. 2002) smile-adjusted implied volatility with the Barone-Adesi & Whaley (1987) early exercise approximation.

    Syntax: sfSABRAmericanCall( alpha, beta, rho, nu, S, K, T, r, q )

    Arg Type Description
    alpha float SABR volatility parameter (α > 0)
    beta float SABR shape parameter (0 ≤ β ≤ 1)
    rho float SABR correlation parameter (-1 ≤ ρ ≤ 1)
    nu float SABR vol-of-vol parameter (ν ≥ 0)
    S float Spot price
    K float Strike price
    T float Time to maturity (years)
    r float Risk-free rate
    q float Continuous dividend yield

    Returns: float — American call option price. Returns Null on invalid input.

    Notes:

  • The forward price F = S·exp((r−q)T) is used internally to compute the SABR IV σ_SABR via the Hagan formula; then the American price is obtained as BAW(S, K, T, r, q, σ_SABR).
  • If q ≤ 0 the early exercise premium is negligible for calls and the price reduces to the European SABR call price.

    • Example:

    
// ATM American call, 5% rate, 2% dividend, 20% base vol, β=0.5

price = sfSABRAmericanCall( 0.2, 0.5, 0, 0.1, 100, 100, 1, 0.05, 0.02 );

    sfAmericanSABROptionPrice

    Multi-asset dispatcher for SABR + BAW American option pricing by underlying type.

    Syntax: sfAmericanSABROptionPrice( type, isCall, alpha, beta, rho, nu, S_or_F, K, T, r, q )

    Arg Type Description
    type float Underlying type: 0 = Spot Equity, 1 = Future, 2 = FX, 3 = Rate Future
    isCall float 1 for call, 0 for put
    alpha float SABR volatility parameter
    beta float SABR shape parameter
    rho float SABR correlation parameter
    nu float SABR vol-of-vol parameter
    S_or_F float Spot price (type 0/2) or futures price (type 1/3)
    K float Strike price
    T float Time to maturity (years)
    r float Risk-free rate
    q float Dividend yield (equity) or foreign rate (FX); unused for futures

    Returns: float — American option price. Returns Null on invalid parameters.

    Notes:

  • For spot/FX (type 0, 2): forward F = S_or_F·exp((r−q)T) is computed internally for SABR IV, then BAW prices using spot.
  • For futures (type 1, 3): S_or_F is used directly as the forward for SABR IV, then BAW76 prices.

    • Example:

    
// American equity call with smile

p1 = sfAmericanSABROptionPrice( 0, 1, 0.2, 0.5, 0, 0.1, 100, 100, 1, 0.05, 0.02 );

// American futures put with smile

p2 = sfAmericanSABROptionPrice( 1, 0, 0.25, 0.7, -0.2, 0.3, 100, 100, 1, 0.05, 0 );

    sfAmericanSABRImpliedVol

    Multi-asset dispatcher that computes the Black-Scholes / Black-76 implied volatility for an American option priced under the SABR-BAW model.

    Syntax: sfAmericanSABRImpliedVol( type, isCall, alpha, beta, rho, nu, S_or_F, K, T, r, q, price )

    Arg Type Description
    type float Underlying type: 0 = Spot Equity, 1 = Future, 2 = FX, 3 = Rate Future
    isCall float 1 for call, 0 for put
    alpha float SABR volatility parameter
    beta float SABR shape parameter
    rho float SABR correlation parameter
    nu float SABR vol-of-vol parameter
    S_or_F float Spot price (type 0/2) or futures price (type 1/3)
    K float Strike price
    T float Time to maturity (years)
    r float Risk-free rate
    q float Dividend yield (equity) or foreign rate (FX)
    price float Market price of the option

    Returns: float — Implied volatility (Black-76 lognormal). Returns Null if parameters are invalid or if no solution exists in [1e-8, 5.0].

    Notes:

  • Uses bisection to find σ such that BAW(S_or_F, K, T, r, q, σ) = price.
  • This is the companion function to `sfAmericanSABROptionPrice`: the implied volatility returned is the Black-76 volatility that would reproduce the given price in the BAW model.

    • Example:

    
// Recover implied vol from an American equity call price

iv = sfAmericanSABRImpliedVol( 0, 1, 0.2, 0.5, 0, 0.1, 100, 100, 1, 0.05, 0.02, 8.5 );
